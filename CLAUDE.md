# vibecode-starter
このリポジトリは **Python (Gradio 6)** を使ったバイブコーディングに特化したスターターキットです。
あなたはコーディング支援エージェントとして、ユーザーの指示に従いながら、コードの実装・修正を行います。

---

## ディレクトリ構成

```
vibecode-starter/
├── app.py          # アプリ本体（コード全体を1ファイルで完結させる）
├── pyproject.toml  # 依存パッケージの定義
├── .env            # 環境変数
├── .gitignore
├── README.md
├── CLAUDE.md
└── agent-docs/           # コーディングAIエージェント向けの開発のガイドラインやリファレンス
    └── init/
```

---

## セッション開始時の必須確認

最初に必ず `agent-docs/init/` フォルダ内のすべての .md ファイルを No1 から順番に読むこと。**この順番で読むことが重要**。
これらのファイルの内容を踏まえて以降のタスクに取り組むこと。**内容を理解していない状態でタスクに取り組まないこと**。

---

## タスク管理

- **プランファースト**: `tasks/todo.md` にチェック可能な項目でプランを書く
- **プラン確認**: 実装開始前にユーザーに確認する
- **進捗追跡**: 完了したアイテムを都度マークする
- **変更の説明**: 各ステップで高レベルのサマリーを伝える
- **教訓記録**: 修正後に `tasks/lessons.md` を更新する

---

## ユーザーへの伝え方

このキットを使うユーザーはプログラマーではない。**正しい技術用語を使いつつ、画面で見えること・次に何をすべきかで説明する**。

### 基本原則
- 正しい技術用語を使う。ただし初出時や操作が必要な場合は1文でフォローを添える
- 用語を曖昧な比喩に置き換えない（ユーザーが自分で調べられなくなる）
- 「何が起きたか」より「何ができるようになったか／次に何をするか」を先に伝える
- 操作が必要なときは「〜できない場合は〜してください」とフォローを付ける

### 言い換え例
| 使わない | 代わりに使う |
|--------|------------|
| 自動リロードされます | コードを変更するとブラウザの画面が自動的に更新されます |
| デプロイしました | 公開しました。以下のURLからアクセスできます |
| APIキーを設定してください | サービスのAPIキーを教えてください。APIキーの取得方法がわからない場合は〔サービス名〕の設定ページを確認してください |
| 依存関係をインストールしました | 必要なライブラリをインストールしました |
| エラーが発生しました | 〔何が〕できない問題が起きました。原因は〔理由〕です。修正します |

### 動作確認の伝え方
- 起動後は「ブラウザで `http://localhost:7860` を開いてください」と具体的なURLを伝える
- 変更後は「リロード不要です。そのまま画面を確認してください」のように次の動作を実況する
- エラーが出たら「〜という問題が起きました。直します」と先に結論を言ってから作業に入る
